## 二、产品概述

### 1. 产品定位

基于广告投放系统底层逻辑设计的SaaS级GEO工具，以"任务"为核心载体，将品牌GEO优化拆解为"目标设定→关键词运营→数据监测→策略落地→效果复盘"的标准化流程，让品牌可按时间周期、任务目标推进GEO优化，实现"每任务有目标、每数据有分析、每优化有闭环"。

### 2. 核心价值

- **流程标准化**：借鉴广告投放"任务化"逻辑，将零散GEO操作转化为"项目创建→执行→监测→优化"的闭环流程，降低操作门槛；
- **关键词智能运营**：支持核心词手动输入 + 系统自动衍生场景词/长尾词，兼容自定义与批量修改，提升关键词覆盖效率；
- **数据对比可视化**：提供"品牌数据 + 竞品数据 + 市场均值"三维对比，让优化方向更精准；
- **策略自动化输出**：基于监测数据自动生成优化策略，配套问题诊断与迭代建议，实现"数据→策略→落地"的快速转化。

### 3. 目标用户

- **品牌方市场部/数字化营销团队**（核心使用者，负责任务创建与优化执行）
- **品牌数据分析师**（负责数据复盘）
- **技术支持**（负责内容部署落地）

### 4. 核心用户操作流程（闭环）

```
项目/任务创建：设定品牌、时间、目标
    ↓
关键词管理：输入核心词→系统衍生场景词/长尾词→自定义/批量修改
    ↓
全程监测启动：推送关键词至全平台监测→生成三维数据报表
    ↓
策略生成：基于报表自动输出优化策略建议
    ↓
优化执行：按策略落地内容调整，同步启动过程监测
    ↓
问题诊断与迭代：监测优化效果→输出问题报告→调整策略/关键词
    ↓
任务复盘，启动新周期任务
```

## 三、核心功能模块详细需求（按用户操作流程排序）

### 模块1：项目与任务管理模块（对应操作流程：项目/任务创建）

#### 1.1 功能名称：项目创建

**功能描述**：支持品牌创建GEO总项目，关联品牌基础信息，作为任务的"容器"。

**核心子功能**：

- **品牌信息配置**
  - 输入品牌名称、所属行业、核心产品（支持多产品关联）、官网地址（用于后续内容部署关联）
  - 信息一旦创建支持修改与补充

- **项目基本设置**
  - 填写项目名称（如"2025Q3家电品牌GEO优化"）
  - 项目周期（开始/结束时间，支持按季度/月度设置）
  - 核心目标（下拉选择：提升AI提及率/强化核心卖点曝光/竞品赶超，可多选并设置优先级）

- **项目成员配置**
  - 支持添加团队成员
  - 按角色分配项目权限（如"负责人可编辑，查看者仅可浏览数据"）

**交互要求**：

- 项目创建页采用"分步引导"设计，第一步填品牌信息，第二步设项目参数，第三步加成员
- 支持项目状态标记（进行中/已结束/暂停）
- 已结束项目可查看历史数据复盘报告

**性能要求**：

- 项目创建响应时间≤2秒
- 单个品牌支持创建≥50个项目
- 项目数据存储期限≥2年

#### 1.2 功能名称：任务创建（核心，借鉴广告投放任务逻辑）

**功能描述**：在项目下创建子任务，明确单任务的时间周期、核心目标与监测范围，作为GEO优化的执行单元。

**核心子功能**：

- **任务基础信息**
  - 关联项目：下拉选择所属总项目
  - 任务名称：按"时间 + 目标 + 范围"格式提示（如"2025.09冰箱品类AI提及率提升任务"）
  - 任务周期：设置具体开始/结束日期（最短7天，最长90天，支持手动调整）
  - 核心目标：细化至可量化指标（如"9月冰箱关键词AI提及率≥80%，超越竞品均值10%"）
  - 监测范围配置：选择任务覆盖的AI平台（默认全选：豆包、文心一言、DeepSeek等，支持手动取消）、分发渠道（官网、知乎、小红书等）
  - 任务负责人：指定1名主负责人 + 多名协作人，支持任务消息定向通知

**交互要求**：

- 任务列表页按"项目分组"展示，支持按"周期/状态/目标"筛选
- 任务详情页显示"目标进度条"（如"提及率目标80%，当前72%"），直观呈现任务完成情况

**性能要求**：

- 单个项目支持创建≥100个任务
- 任务状态更新响应时间≤1秒
- 支持批量导出任务列表（Excel格式）

### 模块2：关键词智能管理模块（对应操作流程：关键词管理）

#### 2.1 功能名称：核心词管理

**功能描述**：支持为任务手动输入核心关键词，作为场景词/长尾词的衍生基础。

**核心子功能**：

- **核心词输入**
  - 手动输入：支持单个输入（如"家用冰箱""节能冰箱"），实时校验是否与品牌/产品匹配（如输入非相关词"手机"，提示"与当前任务'冰箱品类'不匹配，是否继续添加？"）
  - 批量导入：支持Excel模板导入，模板含"核心词、优先级（高/中/低）、关联产品"字段，单次导入≤100个核心词
  - 核心词分类：支持按"产品类型（如冰箱/洗衣机）、功能卖点（如节能/静音）、场景（如家用/租房）"自定义标签，便于后续筛选监测

**交互要求**：

- 核心词列表显示"衍生状态"（如"已衍生场景词12个、长尾词30个"）
- 支持核心词禁用/启用，禁用后关联的场景词/长尾词同步暂停监测

**性能要求**：

- 单个任务支持添加核心词≤50个
- 批量导入响应时间≤3秒
- 关键词匹配校验准确率≥90%

#### 2.2 功能名称：场景词与长尾词智能衍生

**功能描述**：基于核心词自动衍生场景词、长尾词，支持自定义修改与批量操作。

**核心子功能**：

- **自动衍生规则**
  - 场景词：结合AI平台用户提问数据（如核心词"家用冰箱"→衍生"小户型家用冰箱""三口之家家用冰箱"），按"热度（引用频次）"排序
  - 长尾词：结合用户需求痛点（如"家用冰箱怎么选""家用冰箱耗电量多少正常"），按"关联度（与核心词匹配度）"排序

- **衍生词管理**
  - 自定义编辑：支持修改衍生词（如将"小户型家用冰箱"改为"50㎡小户型家用冰箱"），删除无关词（如衍生"商用冰箱"，支持一键删除）
  - 批量操作：支持批量选中衍生词"设为监测词"/"排除"，批量修改标签（如为10个长尾词统一添加"用户问题"标签）
  - 衍生词更新：支持手动触发"重新衍生"（如核心词新增后），或设置"每周自动更新"，补充最新场景词/长尾词

**交互要求**：

- 衍生词列表分"场景词""长尾词"两个标签页，显示"热度值""关联核心词""监测状态"
- 支持输入"自定义衍生方向"（如"围绕'节能'卖点衍生"），系统按方向优化衍生结果

**性能要求**：

- 单个核心词支持衍生场景词≤20个、长尾词≤50个
- 衍生响应时间≤5秒
- 衍生词与核心词关联度≥85%

### 模块3：全平台智能监测模块（对应操作流程：全程监测启动）

#### 3.1 功能名称：关键词全平台监测启动

**功能描述**：任务关键词确认后，一键启动全平台监测，抓取品牌/竞品/市场均值数据。

**核心子功能**：

- **监测触发**
  - 在关键词列表页点击"全程监测"按钮，系统弹窗确认"监测周期（默认与任务周期一致，支持自定义）、监测维度（提及率、权威度、排名）"，确认后推送关键词至监测队列

- **监测进度跟踪**
  - 显示"待监测→监测中→已完成"状态，监测中显示进度条（如"5/8个AI平台已完成抓取"）

- **监测中断处理**
  - 若因AI平台接口问题导致监测中断，系统自动重试（最多3次），重试失败则发送预警通知，标注"未完成监测平台"

**交互要求**：

- 监测启动后，支持在任务详情页查看"实时监测日志"（如"2025-09-05 10:00 完成豆包平台引用数据抓取"）
- 支持手动终止监测（需二次确认），终止后保留已抓取数据

**性能要求**：

- 单次监测覆盖≥8个主流AI平台
- 单个关键词全平台监测完成时间≤30分钟
- 监测成功率≥95%

#### 3.2 功能名称：三维数据报表生成（品牌 + 竞品 + 市场均值）

**功能描述**：基于监测数据自动生成多维度报表，包含关键词基础情况、竞品对比、市场均值参考。

**核心子功能**：

- **关键词基础情况报表**
  - 单词数据：展示每个关键词的"AI提及率（各平台均值）、引用位置分布（如'前3段占比60%'）、权威度评分（提及权威佐证占比）、导流率（引导至官网/咨询量）"
  - 趋势数据：按日/周展示关键词核心指标变化（如"9月1-7日冰箱关键词提及率从65%升至72%"）

- **竞品对比报表**
  - 竞品选择：默认关联任务创建时设置的竞品（最多10个），支持临时添加/删除
  - 对比维度：提及率差值（品牌 - 竞品）、核心卖点引用频次（如"品牌'节能'卖点引用20次，竞品A引用15次"）、权威佐证类型对比（如品牌用SGS报告，竞品用行业协会认证）

- **市场均值报表**
  - 数据来源：聚合所属行业（如家电）、品类（如冰箱）的GEO市场数据，计算"行业均值、头部品牌均值（TOP5）"
  - 差距分析：标注品牌与市场均值的差距（如"提及率72%，行业均值68%，领先4%；权威度评分80分，头部均值85分，落后5分"）

- **报表导出**
  - 支持导出Excel/PDF格式，PDF版含可视化图表（柱状图、折线图）

**交互要求**：

- 报表页支持"关键词筛选""时间周期筛选"（如查看9月1-15日数据）
- 鼠标悬浮至数据差值处，显示"优化建议提示"（如"权威度评分落后5分，建议补充第三方检测报告"）

**性能要求**：

- 报表生成响应时间≤10秒
- 数据误差率≤5%
- 支持同时查看≥20个关键词的对比数据

### 模块4：策略生成与优化执行模块（对应操作流程：策略生成→优化执行）

#### 4.1 功能名称：自动化策略建议生成

**功能描述**：基于三维数据报表，自动输出针对性优化策略，明确"优先级 + 执行步骤"。

**核心子功能**：

- **策略分类**
  - **关键词策略**：如"长尾词'家用冰箱怎么选'提及率仅50%（低于行业均值65%），建议补充'3步选家用冰箱'结构化内容，强化场景匹配"
  - **内容策略**：如"品牌'节能'卖点引用频次低于竞品A（20次 vs 15次），建议在产品页添加'节能认证报告'及'月均耗电量0.5度'数据"
  - **渠道策略**：如"知乎平台提及率（85%）高于小红书（60%），建议在小红书补充'家用冰箱使用场景'图文内容，提升抓取概率"

- **策略优先级**
  - 按"影响程度（高/中/低）+ 执行难度（易/中/难）"划分，高影响 + 易执行的策略排在首位（如"补充权威报告"）

- **策略关联落地**
  - 支持点击策略"生成执行任务"，自动关联至"内容创作模块"或"渠道分发模块"（如点击"补充图文内容"，跳转至内容编辑页）

**交互要求**：

- 策略列表页显示"预期效果"（如"执行该策略后，预计提及率提升8%-12%"）
- 支持策略收藏、分享给协作人，标注"已执行/待执行/放弃"状态

**性能要求**：

- 单个任务支持生成≥20条策略
- 策略生成响应时间≤5秒
- 策略与数据匹配度≥90%

#### 4.2 功能名称：优化过程监测

**功能描述**：跟踪策略执行后的效果变化，实时反馈优化进度，及时发现偏差。

**核心子功能**：

- **监测维度**
  - **指标变化**：按日监测关键词提及率、权威度评分等核心指标，对比优化前后数据（如"优化前提及率72%，优化后第3天升至78%"）
  - **内容落地**：监测优化内容（如补充的报告、图文）是否被AI抓取，显示"抓取状态（已抓取/未抓取）、首次引用时间"
  - **偏差预警**：当优化后指标未达预期（如"执行策略5天后，提及率仅提升2%，未达预期8%"），系统自动发送预警，分析可能原因（如"内容未被AI抓取/竞品同步优化"）

- **优化日志**
  - 记录每次优化操作（如"2025-09-10 补充冰箱节能报告"）及对应效果，形成优化档案

**交互要求**：

- 优化过程页采用"时间轴"展示，左侧为操作记录，右侧为指标变化曲线
- 支持"优化暂停"，暂停后保留当前进度，恢复后继续监测

**性能要求**：

- 优化过程数据更新频率≤2小时/次
- 偏差预警响应时间≤10分钟
- 支持导出优化日志（Excel格式）

### 模块5：问题诊断与迭代模块（对应操作流程：问题诊断与迭代）

#### 5.1 功能名称：优化效果问题诊断

**功能描述**：针对优化后指标未达预期的情况，自动排查问题根源，输出诊断报告。

**核心子功能**：

- **诊断触发**
  - 支持手动发起（在优化过程页点击"问题诊断"）或系统自动触发（偏差预警后1小时内）

- **多维度排查**
  - **内容层**：检查优化内容是否符合AI偏好（如"是否结论前置""是否含可验证数据"）、关键词匹配度（如"内容是否包含目标长尾词"）
  - **技术层**：检查内容Schema标记是否完整、页面加载速度、是否被AI抓取（对接AI平台抓取日志）
  - **竞争层**：分析竞品是否新增优化动作（如"竞品A在9月8日补充节能报告，提及率提升15%"）

- **诊断报告**
  - 标注"问题类型（内容问题/技术问题/竞争问题）、影响程度、具体原因描述"（如"技术问题：补充的节能报告页面Schema标记缺失'数据来源'字段，导致AI未抓取"）

**交互要求**：

- 诊断报告页支持"问题定位"，点击问题条目跳转至对应功能模块（如点击"Schema缺失"，跳转至Schema生成页）
- 支持诊断报告分享，标注"待解决/已解决"状态

**性能要求**：

- 单问题诊断响应时间≤5秒
- 诊断准确率≥85%
- 支持同时诊断≤10个优化问题

#### 5.2 功能名称：迭代策略生成

**功能描述**：基于问题诊断结果，自动生成迭代策略，推动优化效果达标。

**核心子功能**：

- **迭代策略输出**
  - 针对诊断出的问题，提供"具体操作步骤 + 预期效果"
  - 策略按"紧急程度 + 影响范围"排序，优先处理高影响问题

- **迭代执行跟踪**
  - 记录迭代策略执行情况，对比迭代前后数据变化
  - 支持迭代策略效果评估，标注"迭代成功/需继续优化/策略失效"

- **策略模板积累**
  - 将成功迭代策略沉淀为模板，供后续任务复用

**交互要求**：

- 迭代策略页显示"问题→策略→效果"关联视图
- 支持批量执行迭代策略（如同时修复多个技术问题）

**性能要求**：

- 迭代策略生成响应时间≤3秒
- 支持同时处理≤10条迭代策略
- 迭代策略执行跟踪数据更新频率≤1小时/次

## 四、非功能性需求

### 4.1 性能要求

- **系统响应时间**：关键操作响应时间≤2秒，数据查询≤5秒
- **并发支持**：支持≥1000个并发用户同时使用
- **数据处理能力**：单任务支持监测≥500个关键词，报表生成支持≥1000个关键词批量处理

- **操作审计**：记录关键操作日志，支持操作追溯

### 4.3 可用性要求

- **系统可用性**：99.5%以上，年故障时间≤43.8小时
- **数据准确性**：监测数据准确率≥95%，报表数据误差率≤5%
- **兼容性**：支持主流浏览器（Chrome、Safari、Firefox、Edge最新版本）

### 4.4 扩展性要求

## 七、V2.0 迭代升级优化方案

### 7.1 优化核心目标

基于当前 V2.0 版本的 GEO 系统产品架构，聚焦“问题模型构建 - 分析结果 - 策略生成”闭环优化，提升功能之间的业务关联性与操作流畅度。同步匹配最新原型图设计，确保本次迭代后功能逻辑更清晰、路径更短、可操作性更强。

### 7.2 模块名称与按钮调整

- **模块更名**：  
  - 原【数据监测】模块更名为【问题模型构建】，功能定位为“创建查询任务、生成询问语句、发起分析”；  
  - 工作台首页（Dashboard）结构与内容保持不变，仅更新左侧与顶部导航中的模块名称。
- **按钮名称与跳转**：  
  - 【问题模型构建】列表页右上角按钮由【创建监测任务】改为【创建任务】，点击跳转至新的【创建任务】页面；  
  - 列表页中原【查看详情】按钮统一更名为【分析结果】，点击后跳转至分析结果页（原 `monitoring-detail.html`）；  
  - 分析结果页顶部右侧删除【导出页面】【导出 JSON】按钮，新增【创建策略】按钮，用于从当前分析任务直接进入策略创建流程。

### 7.3 【创建任务】页面（问题模型构建）

#### 7.3.1 字段与校验规则

- **基础配置区**  
  - **品牌**：下拉选择，仅展示当前账号授权范围内品牌列表（由管理员在系统设置中预设），必选，单选；  
  - **SKU**：下拉选择，联动品牌，展示所选品牌下的 SKU 列表，必选，单选。
- **核心参数配置区**  
  - **核心词**：文本输入框，支持手动输入 1–5 个核心词（以逗号分隔），用于描述产品核心属性 / 场景，必填；  
  - **询问语句数量**：数字输入框，支持 10–100 的整数（默认 30），必填；  
  - **推荐品牌数量**：数字输入框，支持 3–10 的整数（默认 5），必填；  
  - **相关品牌（可选）**：文本输入框，支持输入最多 5 个竞品品牌名称（逗号分隔），用于增强对比类询问语句生成。

#### 7.3.2 交互逻辑

- **字段完整性校验**：品牌、SKU、核心词、询问语句数量、推荐品牌数量全部合法填写后，【生成询问语句】按钮才可点击，否则置灰并给出缺失提示。  
- **询问语句生成**：点击【生成询问语句】后，系统根据“品牌 + SKU + 核心词 + 相关品牌”综合条件，自动生成适配大模型问答场景的询问语句，并实时展示在下方“询问语句列表”区域。  
- **询问语句编辑**：  
  - 默认全部勾选；  
  - 支持逐条勾选 / 取消勾选与删除无关语句；  
  - 仅对勾选的询问语句发起分析。  
- **发起分析**：用户确认列表后点击【开始分析】按钮，系统创建任务并跳转回【问题模型构建】列表页，对应任务状态显示为“分析中”。

#### 7.3.3 UI 布局要求

- 整体采用“上配置，下结果”的分区结构：  
  - 上方为两个卡片：`基础配置`、`核心参数配置`；  
  - 下方为“询问语句列表”，支持分页（每页 20 条）。  
- 列表项包含：勾选框、询问语句内容、删除按钮。底部展示“已选数量 / 总生成数量”。  
- 必填字段旁统一使用 `*` 标识，标签左对齐，输入框右侧对齐，保证表单一致性。

### 7.4 【分析结果】页面优化

#### 7.4.1 布局与指标

- 保持“三段式”结构：任务基础信息区 + 关键指标区 + 详细结果区，对板块间距与卡片层级做适量放大处理，提升可读性。  
- 指标调整：  
  - 原“目标品牌”指标统一更名为【品牌】，计算逻辑不变；  
  - 新增核心指标【竞品提及率】：
    \[
    竞品提及率 = \frac{相关品牌被提及次数}{目标品牌+所有相关品牌的总提及次数} \times 100\%
    \]
  - 保留并沿用原指标：品牌提及率、品牌平均排名、排名相关性、Top 3 平均分、Top 10 平均分，展示形态为“当前值 + 同比 / 环比变化”。

#### 7.4.2 按钮与行为

- 删除【导出页面】【导出 JSON】按钮，减少干扰操作；  
- 在页面右上角新增【创建策略】按钮：  
  - 点击后跳转到【创建策略】页面；  
  - 自动携带当前分析任务 ID 与关键指标数据；  
  - 作为策略配置的上游数据源。

### 7.5 【创建策略】页面（与分析结果联动）

#### 7.5.1 入口与数据关联

- 入口：仅能从【分析结果】页面点击【创建策略】进入，不支持独立创建。  
- 页面初始化时，系统自动拉取当前任务的关键指标（品牌提及率、品牌平均排名、排名相关性、Top 3 平均分、Top 10 平均分）及行业均值 / 历史表现，用于计算默认目标值。

#### 7.5.2 指标目标值配置

| 指标名称       | 默认目标规则示例                                          |
|----------------|-----------------------------------------------------------|
| 品牌提及率     | 默认目标 = 当前值 × 1.2，支持手动输入 0–100%             |
| 品牌平均排名   | 默认目标 = 当前排名 - 1，支持手动输入正整数             |
| 排名相关性     | 默认目标 = 当前值 + 0.1（上限 1.0），支持 0.0–1.0（1 位小数） |
| Top 3 平均分   | 默认目标 = 当前值 + 0.5（上限 10 分），支持 0–10（1 位小数） |
| Top 10 平均分  | 默认目标 = 当前值 + 0.3（上限 10 分），支持 0–10（1 位小数） |

- 用户可对各目标值逐一调整，页面实时计算并显示“与当前值差值”（如 `+5%`、`-1.2` 等）。

#### 7.5.3 策略建议生成与执行

- 点击【确认策略目标值】后，系统基于：  
  - 当前值与目标值的差距；  
  - 行业均值、历史优秀案例；  
  - 大模型的内容偏好规则；  
  自动生成一份结构化策略建议，包含：  
  - 指标差距优先级（高 / 中 / 低）；  
  - 针对性优化建议（如补充某类询问语句、调整问题模型、强化某类场景）；  
  - 可执行步骤指引（按 1/2/3 步骤拆解）。  
- 用户可选择：  
  - **立即执行**：跳转至具体操作页面（如问题模型构建页，带入部分参数）；  
  - **保存策略**：将本次策略配置与建议写入“策略管理”模块，便于后续复盘与多次执行。
